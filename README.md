# 中国法律法规检索

一款面向 iPhone 和 iPad 的中文法律法规离线浏览与全文检索 App，内置完整的 SQLite 数据库，支持离线使用，并集成 LLM 法律问答功能。

## 功能

### 目录浏览
- 按分类（宪法、法律、行政法规、司法解释等）层级展开浏览 1,902 部法律法规
- 分组支持折叠 / 展开，快速定位目标法律

### 全文检索
- 支持法律名称搜索和条文内容全文搜索
- 搜索结果高亮匹配关键词
- 支持中文数字与阿拉伯数字互转（搜"第10条"自动匹配"第十条"）
- 可选项：
  - **忽略条号匹配**：过滤掉仅因"第X条"编号命中的噪声结果
  - **仅搜标题**：只在法律名称中检索
  - 结果数量上限：50 / 100 / 200 条

### 法律详情
- 完整条文阅读，支持编 / 章 / 节 / 条层级结构展示
- 右侧滚动索引条：按比例显示条号，拖拽或点击直接跳转
- 条内搜索：点击搜索按钮后输入关键词，实时过滤当前法律条文
- 交叉引用：
  - 条文内的法律引用以彩色超链接呈现，点击跳转至被引用条文
  - 每条文底部列出被其他法律引用的来源（被引法律名称 + 条号），可点击跳转

### 法律咨询（AI 问答）

内置两种问答模式，通过底部模式选择器切换：

#### 快速模式（RAG）
基于关键词提取 + 分层检索 + LLM 生成回答的流水线：
1. 问题分解（多问题自动拆分）
2. 领域路由（8 个法律部门）
3. 关键词提取 + 别名扩展
4. 分层检索（法律原文 + 司法解释）
5. 相关性过滤
6. LLM 流式生成回答
7. 参考法条提取（从答案正文引用中匹配）

#### 专家模式
多层专家协作系统，适合复杂法律问题：
- **Coordinator**：接收问题，递进式法律定性，路由至细分专家
- **细分专家**：民法典（合同通则、买卖、租赁、借款、建设工程、服务委托、侵权、产品责任、交通事故、医疗损害、人格权、物权、婚姻家庭、继承等 20+ 专家）、诉讼程序、行政法、刑法等
- **专家组综合**：合并同组专家分析，消除重复
- **Coordinator 整合**：生成最终连贯回答
- 支持追问：当关键事实缺失时，优先从案情文本中自动提取，提取不到才向用户追问（可在设置中配置最多追问轮次）
- 支持并行子问题：多个编号问题（如考试题）并行执行，各自独立回答

#### 通用特性
- 思考过程可展开/折叠，显示每一步的推理内容
- "检索条文"步骤可展开查看具体条文，点击跳转至法律浏览
- 参考法条可展开，点击跳转至对应条文位置
- 对话历史自动保存，可在历史记录中查看、恢复
- 支持多种 LLM 后端（Gemini、DeepSeek、Groq、本地 Ollama），API Key 加密存储于系统 Keychain

### 设置
- 显示/隐藏思考过程
- 专家模式最多追问轮次（0–5）
- 参考法条最多显示数量（0 = 不限）
- 每专家上下文法条数量（5–40）
- 模型选择及 API Key 管理

## 数据库设计

数据库文件 `law_content.db`（SQLite，约 106 MB），只读，随 App Bundle 分发。

### `laws` — 法律元数据

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | INTEGER PK | 自增主键 |
| `title` | TEXT | 法律全称 |
| `category` | TEXT | 分类（宪法、法律、行政法规、司法解释等） |
| `legal_domain` | TEXT | 法律领域 |
| `pub_date` | TEXT | 发布日期 |
| `effective_date` | TEXT | 实施日期 |
| `issuing_org` | TEXT | 发布机关 |
| `doc_number` | TEXT | 发文字号 |
| `total_articles` | INTEGER | 条文总数 |
| `is_current` | INTEGER | 是否现行有效（1 = 是，0 = 已废止或被修订版本取代） |

### `nodes` — 条文层级节点

每部法律的内容被解析为树形结构，`type` 字段区分节点层级：

| type 值 | 含义 |
|---------|------|
| `part` | 编 |
| `chapter` | 章 |
| `section` | 节 |
| `article` | 条文 |

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | INTEGER PK | 节点 ID |
| `law_id` | INTEGER FK | 所属法律 |
| `parent_id` | INTEGER FK | 父节点（树形结构） |
| `type` | TEXT | 节点类型 |
| `article_number` | TEXT | 条号文本（如"第一条"） |
| `content` | TEXT | 节点完整文本 |
| `global_order` | INTEGER | 全局排序序号 |
| `article_num` | INTEGER | 条号数值（阿拉伯数字，用于跳转定位） |

**索引**：`(law_id, global_order)`、`(law_id, part_num, chapter_num, section_num, article_num)`

### `nodes_fts` — 全文检索虚表（FTS5，trigram 分词）

```sql
CREATE VIRTUAL TABLE nodes_fts USING fts5(
    content,
    content_body,   -- 去除条号前缀后的正文，用于"忽略条号匹配"模式
    article_number,
    tokenize='trigram'
);
```

trigram 分词支持任意中文子串匹配，无需分词词典。另有 `nodes_fts_bigram`（unicode61 分词）用于短词（少于 3 个汉字）的二元检索。

### `article_references` — 条文引用关系

| 字段 | 说明 |
|------|------|
| `from_*` | 引用来源（法律 ID、条号等） |
| `to_*` | 被引用目标 |
| `ref_type` | `cross_law`（跨法引用）或 `self_ref`（本法内引用） |
| `raw_text` | 原文中的引用文字（用于超链接定位） |
| `resolved` | 是否成功解析到具体节点 |

共 4,994 条引用记录。

## 数据来源

法律原文数据来源于 [法律快车 · 法律法规库](https://github.com/flowerwind/chinese-law) 及中国政府官方网站公开发布的法律法规全文。

中华人民共和国法律法规原文属于国家公开文献，不受著作权保护，可自由使用。

## 技术栈

- **语言**：Swift 6
- **UI 框架**：SwiftUI
- **数据库**：SQLite（通过 C API 直接调用，不依赖 ORM）
- **LLM 集成**：Gemini Flash、DeepSeek、Groq（Llama）、Ollama（本地）
- **最低系统要求**：iOS 17 / iPadOS 17
- **支持设备**：iPhone、iPad（含旋转适配）
